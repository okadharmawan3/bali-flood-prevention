# Bali Flood Labeler

Use this skill to label sample folders created by `scripts/generate_samples.py`.
Each sample folder contains:

```text
rgb.png
swir.png
metadata.json
```

Write exactly one `annotation.json` next to those files. The annotation must be a clean JSON object only, with no markdown, comments, captions, or wrapper metadata.
Use Codex's current vision reasoning to inspect the images directly. Do not call Anthropic or any external labeling API.

## Batch workflow

1. Read `data/{run_id}/manifests/unlabeled.jsonl` or `missing_labels.jsonl`.
2. Work in small batches so image inspection stays reliable.
3. For every row, open both images:
   - `rgb.png`: natural-color context for settlements, roads, rivers, coast, fields, and visible water.
   - `swir.png`: water and saturated surfaces are usually darker than dry land; wet vegetation or soil may look muted compared with dry vegetation.
4. Read `metadata.json` for region, point id, timestamp, tile center, and size.
5. Write `annotation.json` with the exact schema below.
6. Run `uv run scripts/check_samples.py {run_id}` after each batch to refresh `labeled.jsonl` and `missing_labels.jsonl`.

## Exact schema

```json
{
  "flood_risk_level": "low",
  "water_extent_level": "small",
  "standing_water_present": false,
  "temporary_inundation_likely": false,
  "urban_or_infrastructure_exposure": false,
  "road_or_transport_disruption_likely": false,
  "cropland_or_settlement_exposure": false,
  "river_or_coastal_overflow_context": false,
  "low_lying_or_poor_drainage_area": false,
  "vegetation_or_soil_saturation": false,
  "permanent_water_body_present": false,
  "cloud_shadow_or_image_quality_limited": false,
  "confidence": "medium"
}
```

Allowed enum values:

- `flood_risk_level`: `low`, `medium`, `high`
- `water_extent_level`: `small`, `moderate`, `large`
- `confidence`: `low`, `medium`, `high`

All other fields are booleans.

## Labeling rules

- Use `high` flood risk when standing water or probable inundation is extensive and affects, surrounds, or directly threatens settlements, roads, bridges, cropland, or other infrastructure.
- Use `medium` flood risk when water or saturation is visible but exposure is limited, ambiguous, or mostly near natural drainage paths.
- Use `low` flood risk when no abnormal standing water is visible, water is limited to normal rivers, coastlines, reservoirs, ponds, or the image is mostly dry/upland.
- Set `temporary_inundation_likely` only when visible water appears abnormal for the scene, such as water spread across fields, roads, floodplains, urban blocks, or away from normal channel boundaries.
- Set `permanent_water_body_present` for normal rivers, lakes, reservoirs, ponds, wetlands, fishponds, or coastline, even when flood risk is low.
- Set `river_or_coastal_overflow_context` when water is spatially connected to rivers, drainage channels, deltas, reservoirs, wetlands, or the coast.
- Set `low_lying_or_poor_drainage_area` when the scene appears flat, floodplain-like, coastal, deltaic, paddy-field dominated, or poorly drained. Lower confidence if this is inferred only from imagery.
- Set `vegetation_or_soil_saturation` when SWIR/RGB suggests wet vegetation, saturated soil, muddy surfaces, or moisture-stressed land.
- Set `cloud_shadow_or_image_quality_limited` when clouds, shadows, haze, no-data, poor illumination, or artifacts limit reliable interpretation.
- Use `confidence: "low"` when clouds, ambiguity, coastal/permanent-water confusion, or low image quality substantially limits certainty.

## Output discipline

Write only this file for each label:

```text
annotation.json
```

The file must contain only valid JSON matching the exact schema. Do not include explanations in the file. If you need to explain uncertainty, reflect it through the fields and `confidence`.
